# WordPressテーマ「SWELL」前提：アスティア (ASTIA) Webサイト ワイヤーフレーム設計書

本ワイヤーフレームは、WordPressテーマ「**SWELL**」で実装しやすいWebサイト設計を目的とした**第1段階（骨子作成フェーズ）**のプロトタイプです。
ブラウザで各HTMLを開くことで、**デスクトップ画面（1440px想定／最大幅1200px中央寄せ）**におけるレイアウト、セクション順、見出し、CTA、導線、カード配置の大枠をインタラクティブにご確認いただけます。

---

## 1. デザイン＆実装ルール概要

| 項目 | 採用ルール | SWELL実装時のポイント |
| :--- | :--- | :--- |
| **画面解像度** | Desktop Only (1440px想定) | 1カラム構成を基本とし、コンテナ最大幅を `1200px` に設定 |
| **カラーパレット** | 白・黒・グレーのモノクロ仕様 | グレースケールで骨格を視覚化。実際のサイトではブランドカラーへ差し替え可能 |
| **装飾・背景** | 最小限（装飾過多・グラデーション廃止）| SWELLの「フルワイドブロック」「背景色（薄いグレー）」を活用してセクションを区切る |
| **画像・アイコン** | すべてグレーのプレースホルダー表示 | 各プレースホルダーには格納されるべき写真・図解の意味をラベル表記 |
| **テキスト・文章** | 短いラベルおよびダミーテキスト | 本文を入れ込まず、ページ全体の役割やストーリー展開の確認を優先 |
| **共通パーツ** | ヘッダー・フッターのコンポーネント化 | サイト全体で共通ナビゲーションおよびお見積もりCTAへの導線を統一 |

## 3. WordPressテーマ「SWELL」ブロック再現ガイド

各ページで採用しているレイアウトは、SWELLの標準機能・ブロック構成のみで簡単に再現可能です。

### 📌 2カラム構成（テキスト ＋ 画像／動画）
* **対象箇所**：トップ「アスティアとは」、製品紹介「製品概要」、会社概要「開発に込めた想い」など
* **SWELL再現方法**：
  * 「**カラムブロック（50 : 50）**」または「**メディアとテキストブロック**」を使用
  * カラム間の余白（ギャップ）は「広め」または「40px〜50px」に設定
  * 縦方向の配置は「中央揃え」に指定することで Auto Layout に近い整列を実現

### 📌 3カラム・カードグリッド
* **対象箇所**：トップ「6つの特徴」、製品紹介「9つの設置場所」、導入事例「事例一覧」
* **SWELL再現方法**：
  * 「**リッチカラムブロック（3列）**」または「**投稿リストブロック（カード型／3列）**」を使用
  * 特徴カード・設置場所カードは「**グループブロック**」または「**キャビネットブロック**」で囲み、背景色を白・ボーダーを細いグレーに設定
  * 導入事例カードはカスタム投稿または通常投稿で作成し、SWELLの「**投稿リスト（カード・3カラム・バッジ表示あり）**」で出力

### 📌 テーブル（導入方法比較表・会社概要情報）
* **対象箇所**：製品紹介「導入方法比較表」、導入事例詳細「基本情報」、会社概要「会社情報・ヤマゲン情報」
* **SWELL再現方法**：
  * 「**テーブルブロック**」を使用
  * 表のスタイル設定で「**ストライプ（1行おきに色替え）**」を有効化
  * ヘッダーセルの背景色を「薄いグレー」、文字色を「黒・太字」に設定
  * セル内余白（パディング）は CSS または SWELL 設定で広めにキープ

### 📌 FAQアコーディオン
* **対象箇所**：よくある質問「全6カテゴリ」
* **SWELL再現方法**：
  * SWELL専用「**アコーディオンブロック**」を使用
  * アイコン表示を「**Q/Aアイコンあり（または丸・四角バッジ）**」に設定
  * 質問文の背景に「薄いグレー」のアクセントカラーを適用

### 📌 ページ誘導バナー・CTAブロック
* **対象箇所**：各ページ下部・トップの各製品・事例・FAQバナー
* **SWELL再現方法**：
  * SWELL専用「**バナーリンクブロック**」または「**フルワイドブロック ＋ グループブロック（余白大・角丸・ボーダー有）**」を使用
  * ボタンは SWELL標準「**SWELLボタン（立体・キラリ・ノーマル）**」からシンプルなグラデーションなしスタイルを選択

## 4. 今後のフェーズに向けたチェックポイント

1. **セクション順・レイアウトの承認**
   * この骨子ワイヤーフレームで、ユーザーが欲しい情報にスムーズにたどり着ける導線になっているかをご確認ください。
2. **コピーライティングと本文・FAQの流し込み**
   * ワイヤーフレーム合意後、各セクションのダミーテキスト部分に実際の製品スペック、解決事例の具体的なインタビュー、FAQの正式な回答文を流し込みます。
3. **ビジュアル素材・図解デザインの制作**
   * 「循環システムの仕組み図解」や「導入写真」「製品外観写真」を制作・撮影し、グレーのプレースホルダーへ入れ替えていきます。
